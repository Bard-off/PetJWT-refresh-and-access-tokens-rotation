# Multi-Device Session Manager (FastAPI + Redis)

Реализация продвинутой системы аутентификации с управлением активными сессиями на разных устройствах, ротацией токенов и возможностью дистанционного завершения сеансов.

## 🎯 О проекте
Стандартные JWT-токены обычно являются "stateless", что затрудняет их мгновенный отзыв. Данный проект решает эту проблему, добавляя серверный контроль над сессиями (Stateful JWT) без потери производительности, характерной для классических сессий в БД.

**Ключевая ценность:** Пользователь может видеть список всех своих входов (устройство, IP, время) и принудительно завершить сессию на другом устройстве (например, при утере телефона).

## 🚀 Основной функционал
* **Stateful JWT (RS256):** Использование пар ключей (публичный/приватный) для повышения безопасности.
* **Refresh Token Rotation:** При каждом обновлении Access-токена выдается новый Refresh-токен. Старый мгновенно аннулируется.
* **Detection of Reuse:** Если скомпрометированный (старый) Refresh-токен используется повторно, система блокирует всю цепочку сессий данного пользователя.
* **Session Management:**
    * `GET /auth/sessions` — список активных устройств (User-Agent, IP, дата входа).
    * `DELETE /auth/sessions/{session_id}` — отзыв конкретной сессии по ID.
    * `DELETE /auth/sessions/logout-all` — завершение всех сеансов, кроме текущего.
* **Whitelist в Redis:** Проверка валидности `jti` (JWT ID) через быструю In-memory базу данных.

## 🛠 Технологический стек
* **Framework:** FastAPI
* **Database:** PostgreSQL (SQLAlchemy 2.0) — хранение данных пользователей.
* **Cache/Session Store:** Redis — хранение метаданных сессий и белого списка токенов.
* **Auth:** PyJWT (RS256), Passlib (bcrypt).
* **Containerization:** Docker + Docker Compose.

## 🧠 Реализованные концепции (Middle Skills)
1.  **Инвалидация токенов:** Механизм мгновенного отзыва доступа до истечения срока действия JWT.
2.  **Работа с метаданными:** Парсинг `User-Agent` для отображения понятного названия устройства (например, "Chrome on MacOS").
3.  **Безопасность:** Защита от Replay-атак с помощью уникальных идентификаторов токенов (`jti`).
4.  **Middleware:** Слой проверки каждой сессии в Redis перед допуском к защищенным эндпоинтам.
